# Совместимость с Windows 10

Этот проект разрабатывается на `macOS`, но целевая среда запуска — `Windows 10`. Ниже зафиксированы правила, которые стоит соблюдать с самого начала.

## 1. Пути к файлам

- Не собирать пути строками вроде `"data/" + filename`.
- Использовать `pathlib.Path`.
- Не хардкодить разделители `/` и `\`.

Пример:

```python
from pathlib import Path

input_path = Path("data") / "example.ndax"
output_path = Path("output") / "result.csv"
```

## 2. Запуск без shell-зависимостей

- Не завязывать решение на `bash`, `grep`, `sed`, `pwd`, `./script.py`.
- Вся основная логика должна запускаться обычной командой Python.
- В документации давать примеры запуска для Windows, например:

```bash
py script.py
```

## 3. Кодировки и кириллица

- Всегда явно указывать кодировку при чтении и записи текстовых файлов.
- Для CSV, который будут открывать в Excel на Windows, использовать `encoding="utf-8-sig"`.
- Если используется модуль `csv`, открывать файл с `newline=""`.

Пример:

```python
with open("table.csv", "w", encoding="utf-8-sig", newline="") as f:
    ...
```

## 4. CSV для Excel

- Проверить, какой разделитель удобнее для целевой среды: часто на Windows/Excel лучше работает `;`.
- Если таблица делается через `pandas`, задавать параметры явно:

```python
df.to_csv("table.csv", sep=";", index=False, encoding="utf-8-sig")
```

## 5. Графики и шрифты

- Не использовать редкие системные шрифты.
- Проверить, что подписи осей и легенда корректно отображаются на Windows.
- Перед релизом открыть сохраненный `jpg` именно на Windows-машине.

## 6. Проверки перед использованием

- Протестировать запуск в чистой среде `Windows 10`.
- Проверить чтение `example.ndax`.
- Проверить открытие `csv` в Excel.
- Проверить, что `jpg` строится без ошибок и с читаемыми подписями.

Главный принцип: писать решение так, чтобы оно зависело только от `Python` и выбранных библиотек, а не от особенностей терминала или оболочки на `macOS`.
