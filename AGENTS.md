# AGENTS.md

В этом файле фиксируются только repo-specific правила работы в этом репозитории. Общие инженерные практики и стандартные навыки агента здесь не дублируются.

## Что делать перед реализацией любой фичи

- Сначала продумай детали реализации.
- Если по реализации есть неоднозначности, задай пользователю уточняющие вопросы.
- После получения ответов зафиксируй уточнения и ограничения.
- Затем декомпозируй большую задачу на маленькие изолированные подзадачи в рамках плана.

## Планирование и распараллеливание

- По возможности отдавай план на реализацию субагентам в максимально распараллеленном режиме: одновременно может работать до 6 субагентов.
- Параллелить можно только независимые подзадачи. Зависимые подзадачи запускай только после завершения предыдущих.
- Декомпозируй подзадачи так, чтобы они минимально пересекались по коду (чтобы можно было максимально распараллелить).
- До начала реализации заранее установи порядок мержа веток.
- В плане явно отмечай, какие подзадачи можно выполнять параллельно, какие зависят от предыдущих, и в каком порядке ветки должны мержиться в `master`.
- Формат записи зависимостей в плане: `задача1 -> задача2 + задача3 + задача4 -> задача5`

## Правило работы над одной подзадачей

- Сначала запускай субагента-кодера.
- Субагент-кодер должен создать отдельную ветку, название которой отражает подзадачу, внести изменения и сделать commit.
- После этого запускай субагента-ревьюера с релевантными skills.
- Субагент-ревьюер должен составить список замечаний и вернуть их субагенту-кодеру.
- Повторяй цикл `coder -> reviewer` до тех пор, пока ревьюер не подтвердит результат.
- После этого аккуратно мержи ветку подзадачи в `master`.

## Правило для commit message

- Каждый коммит должен быть лаконичным и осмысленным. Он должен четко отражать характер изменений, которые были сделаны, не вдаваясь в детали. Я, прочитав текст коммита, должен ответить сам на один из вопросов: "что изменилось?", "что улучшилось?", "от какой проблемы избавились?"
