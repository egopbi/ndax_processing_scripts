# Идея проекта
Проект нужен для автоматизации рутины на работе в химической лаборатории, занимающейся исследованиями в области электрохимических свойств аккумуляторов

Работники хотят на основе NDAX-файлов (один файл - один образец) строить сравнительные графики и делать показательные сравнительные таблицы. Сравнивают образцы

ВАЖНЫЙ НЮАНС: рабочая среда - windows 10. Соответственно, нужно выбрать удобный для реализации этой задачи язык (можно питон, но это необязательно), при этом по возможности нужно устанавливать как можно меньше тяжелых зависимостей. Для такой задачи поднимать докер-контейнер будет нерационально


# Реализация
Проект включает в себя 2 скрипта:
1. Построение графика по выбранным образцам по выбранным величинам
2. Построение сравнительной csv-таблицы по выбранным образцам по выбранным величинам (просто срез таблицы)


## Скрипт отрисовки графика
Скрипт принимает несколько путей до файлов с образцами (как я уже говорил, один файл - один образец), принимает явные названия величин, которые пользователь хочет получить на графике. То есть пользователь явно выбирает образцы и выбирает координатные оси x и y (x по дефолту время, но опционально это можно изменить). На выходе пользователь получает график формата jpg. Пример такого графика приведен в `example_photo.jpg`.

Также опционально пользователь еще одним аргументом может указать диапазон значений для осей OX и OY (по дефолту график выводится по всему деиапазону, который есть в файле). Например, пользователь может указать, что он хочет видеть график по двум образцам в координатах напряжение от времени в следующих диапазонах: время - от 40 до 200, напряжение - от -50 до +50


Обрати внимание, что всегда оси должны быть подписаны так: <величина> (<единица_измерения>). В правом верхнем углу должна быть легенда - соответствие цвета названию образца


## Скрипт создания сравнительной таблицы
Скрипт принимает несколько путей до файлов с образцами, принимает название величин, по которым будет сравнение. На выходе из скрипта должна быть сравнительная таблица формата csv. Пример таблицы приведен в `example_table.csv`
